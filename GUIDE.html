Nano GDPR

Cookie consent for static websites. No database. No server. Free. MIT.

What it does

It shows a cookie banner. The visitor picks Accept or Reject. Their choice is saved in their own browser. Your tracking scripts only run if they say yes.

Step 1 - Add it to your page

Put the files in a folder called gdpr on your website (the same way your shop and blog have their own folder). Then pick one option and put it in the page <head>.

Option A - set the words on the page. Use this if you want to change the text here:

<link rel="stylesheet" href="/gdpr/nano-gdpr.css">

<script>
  window.NanoGDPR = {
    text: "We use cookies to improve your experience.",
    privacyUrl: "/privacy-policy"
  };
</script>

<script src="/gdpr/nano-gdpr.js" defer></script>

Option B - just the script. One line. The words are set inside the file instead:

<script src="/gdpr/nano-gdpr.js" defer></script>

The banner shows itself. You do not paste any banner code.

Step 2 - Hold back your tracking scripts

This is the important bit. Find your tracking script (like Google Analytics). Change type to text/plain and add a label. Now it waits for consent.

Before (runs straight away - not allowed):

<script src="https://www.googletagmanager.com/gtag/js?id=G-XXXX"></script>

After (waits until the visitor says yes):

<script type="text/plain" data-nano-gdpr="analytics"
        data-src="https://www.googletagmanager.com/gtag/js?id=G-XXXX"></script>

If your script is inline, just wrap it the same way:

<script type="text/plain" data-nano-gdpr="analytics">
  // your tracking code here
</script>
The label analytics matches the Analytics toggle in the popup. There is also a functional label. Essential cookies are always allowed.

Step 3 - Add a "Cookie settings" link

So people can change their mind later. Put this anywhere, like your footer:

<a href="#" data-nano-gdpr-open>Cookie settings</a>

Optional - keep a record (the log)

By default, choices are saved in the visitor's browser only. There is no record on your side.

If you want a record (proof), turn on the log. This needs PHP and Apache hosting. It will not work on a pure static host.

1. The optional-logging folder is already inside your gdpr folder. It holds log.php and a .htaccess file. Just upload it with the rest.

2. In your settings, add this line:

  logUrl: "/gdpr/optional-logging/log.php"

Now each choice is added to a log file inside that folder. The .htaccess file keeps the log private.

If you do nothing, the log stays off and the banner still works fine.

Settings you can change

  • text - the message in the banner
  • privacyUrl - link to your privacy page
  • position - "bottom" or "top"
  • colors - { bg, text, accept, reject, link }
  • categories - your own list of toggles
  • logUrl - turn on the record log (needs PHP). Leave empty for off.

Try it

Open demo/index.html in a browser. Open the console (press F12). Click Accept - the test script runs. Click the Reset button to see the banner again.

Where is the data?

Only in the visitor's browser. Nothing is sent anywhere. There is no server, no database, no login, nothing to hack.
